# 搜索策略

## 核心原则

公司调研最容易翻车的地方是**信息过时**和**信息源不可靠**。这份文档说明如何最大化搜索效率和准确性。

## 搜索的三层结构

### 第一层：建立公司画像（3-4 个 query）

目标：快速搞清楚这家公司是谁、做什么、规模多大。

```
"公司全名" 介绍 总部 成立
"公司全名" CEO 创始人
"公司全名" 营收 财报 员工
"公司全名" 业务线
```

如果是上市公司，可直接搜年报或财报。如果是大厂子品牌（如荣耀之于华为系），要单独搜清楚母公司关系和独立运营时间点。

### 第二层：业务+产品+动态（5-7 个 query）

```
"公司全名" 主要产品
"公司全名" 战略 2025
"公司全名" 新品发布 2025
"公司全名" CEO 演讲 战略发布
"公司全名" 组织架构 调整 2025
"公司全名" 投资 收购 2025
```

时间约束很重要：搜索 query 里带"2025"或"2026"能过滤掉早期信息。

### 第三层：岗位定位（3-4 个 query）

针对 JD 里提到的关键业务/产品深挖：

```
"公司全名" "JD 核心产品名" 功能
"公司全名" "JD 核心产品名" 用户规模
"公司全名" "JD 核心产品名" 竞品对比
"行业关键词" 玩家格局 2025
```

例：JD 提到"大模型办公场景"，就搜 `荣耀 YOYO 办公 PPT`、`荣耀 一语生成 PPT`、`手机 AI 助手 办公场景 玩家`。

## 信息源的可信度分级

| 级别 | 来源 | 使用方式 |
|---|---|---|
| S | 公司官网、官方新闻稿、上市公司公告、CEO 公开演讲实录 | 直接引用，标注"官方" |
| A | 头部财经媒体（财新、第一财经、36Kr 深度报道）、行业研究报告 | 引用并标注媒体名 |
| B | 一般科技媒体、自媒体深度文章 | 交叉验证后引用 |
| C | 普通博客、社交媒体 | 仅作线索，不直接引用 |

## 关键质量动作

### 1. 时间戳必查

搜索结果里每条信息必须确认日期。WebSearch 返回结果通常带发布日期，要看清楚。**报告里所有动态信息必须带具体日期**（YYYY 年 MM 月），不允许"最近""目前"。

### 2. 数字交叉验证

涉及营收、用户量、市场份额这类数字，**至少两个独立来源验证**。如果只有一个来源且无法验证，标注"信息来源单一，待验证"。

### 3. 区分官方表态和媒体解读

例：荣耀说"投入 100 亿美元建设 AI 终端生态"——这是官方表态。媒体解读"荣耀 All in AI"——这是媒体加工。报告里要区分。

### 4. 同名公司风险

如"小米"可能指小米集团、小米汽车、米家等多个主体。如"理想"可能是理想汽车也可能是其他公司。**搜索前先用第一轮 query 锁定公司主体**，看清楚是哪家公司、上市代码是什么。

## 搜索失败时怎么办

如果搜索结果稀疏（这种情况会出现在小公司、未上市公司、刚成立的初创团队）：

1. 换关键词角度搜：搜创始人个人、搜产品名、搜行业关键词
2. 搜招聘信息：BOSS、脉脉、领英上的公司动态有时是唯一信息源
3. 搜投融资数据库（IT 桔子、烯牛数据等）
4. 直接告诉用户："这家公司公开信息较少，建议你通过认识的人了解 / 留意面试现场观察"，而不是硬凑内容

诚实比虚构有价值。
